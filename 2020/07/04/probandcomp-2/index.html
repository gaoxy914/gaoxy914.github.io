Application: Coupon Collector’s Problem

Each box of cereal contains one of $n$ diffrent coupons. Assuming that the coupon in each box is chosen independently and uniformly at random form the $n$ possibilities, how many boxes of cereal must you buy before you obtain at least one of every type of coupon?

Let $X$ be the number of boxes bought until at least one of every type of coupon is obtained and $X_{i}$ be the number of boxes bought for a new coupon while you had exactly $i - 1$ different coupons, then $X = \sum^n_{i = 1}X_i$.

When exactly $i - 1$ coupons have been found, the probability of obtaining a new coupon is $p_i = 1 - \frac{i - 1}{n}$, $X_i$ is a geometric random variable with parameter $p_i$, $E[X_i] = \frac{1}{p_i} = \frac{n}{n - i + 1}$. $E[X] = n\sum^n_{i=1}\frac{1}{i}$.

Lemma The harmonic number $H(n) = \sum^n_{i=1}1/i$ satisfies $H(n) = \ln{n} + \Theta(1)$.
For the coupon collector’s problem, the expected number of random coupons required to obtain all $n$ coupons is $n\ln{n} + \Theta(n)$.

Application: The Expected Run-Time of Quicksort

Quicksort is a simple and in practice very efficient sorting algorithm. A call to the Quicksort function begins by choosing a pivot element $x$ from the set. The algorithm proceeds by comparing every other element to $x$, dividing the list of elements into two sublists: those that are less than $x$ and those that are greater than $x$. Then Quicksort recursively sorts these sublists.

In the worst case, Quicksort require $\Omega(n^2)$ comparison operations. However, a resonable choice of pivots would require many fewer comparisons. For example, if pivot always split the list into two sublists of size at most $\lceil n/2 \rceil$, the number of comparisons is $O(n\log{n})$.

First, We can change the algorithm to choose the pivots randomly.
Second, we can keep our deterministic algorithm, but consider a probabilistic model of the inputs.(This yields a probabilistic analysis based on a model of the inputs)

Random Quicksort:

Theorem Suppose that, whenever a pivot is chosen for Random Quicksort, it is chosen independently and uniformly at random from all possibilities. Then, for any input, the expected number of comparisons made by Random Quicksort is $2n\ln{n} + O(n)$.
Proof: Let $y_1, y_2, \cdots, y_n$ be the same values as the input values $x_1, x_2, \cdots, x_n$ but sorted in increasing order.

Then the total number of comparisons $X = \sum^{n-1}_{i=1}\sum^n_{j=i+1}X_{ij}$.

$y_i$ and $y_j$ are compared if and only if either $y_i$ or $y_j$ is the first pivot selected by Random Quicksort from the set $\{y_i, y_{i+1}, \cdots, y_{j-1}, y_j\}$, i.e., $\mathrm{Pr}(X_{ij} = 1) = \frac{2}{j - i + 1}$.
$E[X] = (2n + 2)\sum^n_{k=1}\frac{1}{k} - 4n$.

Deterministic Quicksort

Theorem Suppose that, whenever a pivot is chosen for Quicksort, the first element of the sublist is chosen. If the input is chosen uniformly at random from all possible permutations of the values, then the expected number of comparisons made by Deterministic Quicksort is $2n\ln{n} + O(n)$.
Proof: Agin, $y_i$ and $y_j$ are compared if and only if either $y_i$ or $y_j$ is the first pivot selected by Quicksort from the set $\{y_i, y_{i+1}, \cdots, y_{j-1}, y_j\}$. Since all possible permutations of the input values are equally likely, every element is equally likely to be first.
